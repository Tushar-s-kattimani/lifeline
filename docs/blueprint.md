# **App Name**: LifeLine Heatmap

## Core Features:

- Real-time Donor Heatmap: Display a Google Maps heatmap showing blood donor availability in real-time.
- Emergency Blood Request: Enable users to send one-tap emergency blood requests, including blood type, hospital, and urgency details.
- Donor Availability Toggle: Allow donors to toggle their availability status, indicating whether they are currently able to donate.
- Donor Matching and Notification: Use a tool to match blood requests with eligible nearby donors, sending instant alerts using Firebase Cloud Messaging.
- User Authentication: Implement Firebase Authentication for user login/signup via phone number or Google account.
- Donor Profile Management: Enable donors to manage their profiles, including blood group, last donation date, and contact information.

## Style Guidelines:

- Primary color: Soft red (#E63946) to evoke a sense of urgency and compassion.
- Background color: Light pink (#F1FAEE), a heavily desaturated hue to soften the appearance of the red while remaining tonally aligned.
- Accent color: Pale orange (#F4A261) to highlight key CTAs.
- Headline font: 'Playfair' serif for titles and important headings to bring a touch of modern elegance; body font: 'PT Sans' for a clean, readable user experience.
- Use minimalist, custom icons for navigation and feature representation to convey the important actions without overwhelming users, especially in emergency situations.
- Employ rounded cards and floating buttons for a modern, approachable interface with a clear focus on the 'Need Blood Now' CTA.
- Implement subtle animations and micro-interactions for user feedback and engagement, particularly on the splash screen (heartbeat) and when sending/receiving blood requests.
